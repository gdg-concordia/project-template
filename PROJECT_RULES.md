# Project General Rules

1. All projects must be open source.
2. Every project must have 1+ core maintainer.
3. All changes must go through pull requests. Direct pushes to main are not allowed.
4. All new projects must use this template repository.
5. Projects must stay active. Inactive projects should be archived.

# Project Setup Rules

1. Every new project **must create and update milestones** to track key phases (e.g., setup, basic features, first release).  
2. Every new project **must create X starter issues** labeled `good first issue` or `help wanted` to help new contributors get started quickly.
